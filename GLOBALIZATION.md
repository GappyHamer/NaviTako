# 글로벌라이즈(다국어) 적용 정리

타코쨩(TTakochan)의 다국어 구조와 **언어별 번역 방식**을 정리한 문서입니다.

## 1. 아키텍처 (라우팅 i18n)

- **라이브러리**: `next-intl` (App Router)
- **URL 방식**: `localePrefix: "as-needed"` — 한국어는 접두사 없이 `/`(기존 URL 유지), 나머지는 `/{locale}`.
  - 예) `/`(ko), `/en`, `/ja`, `/ru/market`, `/es/guide/...`
- **로케일 11개**: `ko`(기본), `en`, `ja`, `ru`, `uk`, `vi`, `hi`, `tr`, `pt`, `id`, `es`
- **언어 스위처**: 우상단(`components/LangSwitcher.tsx`) — 현재 경로를 유지한 채 언어만 전환.
- **SEO**: 레이아웃 `generateMetadata`에 hreflang(11개 + `x-default`), `sitemap.xml`에도 언어별 URL. 로케일별 `<title>`/`description` 번역.
- **메시지 구조**: `messages/{locale}/{namespace}.json`
  - `common` — 헤더 내비·푸터·언어명·메타 제목/설명·면책·홈 알고리즘 섹션
  - `oracle` — 예언 버튼·쿨다운·바로받기·광고모달·결과 UI
  - `pages` — 시장온도·소개·예언하기·개인정보·문의·면책·가이드목록 본문
  - `fun` — 히어로 회전문구·로딩문구·결과 키워드 풀(주제/센티먼트/타깃)
- **폴백**: `i18n/request.ts`가 **ko를 베이스로 깔고** 요청 로케일 번역을 덮어씀 → 번역 누락 키는 자동으로 한국어로 표시(크래시 없음).
- **가이드 글**: `content/guide/{locale}/{slug}.md` (없으면 ko로 폴백, `lib/guide.ts`).

## 2. 번역 원칙

- **UI/본문(oracle·pages·common)**: 의미 보존 자연 번역. 플레이스홀더(`{n}`,`{h}`,`{m}`,`{s}` 등)와 리치텍스트 태그는 그대로 유지.
- **재미 콘텐츠(fun: 히어로문구·로딩·키워드)**: **직역 금지.** 각 나라 **암호화폐 커뮤니티에서 실제로 쓰는 밈·구어체**로 새로 창작(개수·톤은 한국어 원본 참고). 한국 특유 표현(김치프리미엄·서학개미·흑우 등)은 각국 정서에 맞게 치환.

## 3. 언어별 적용 방식

| 언어 | 대상 국가 | 톤/접근 | 예시 |
|---|---|---|---|
| **ko** 한국어 | 한국 | 원본 (기준) | 떡상, 존버, 김치코인 |
| **en** English | 미국·나이지리아·인도영어권·필리핀·글로벌 | 캐주얼 크립토 커뮤니티 영어 | "Get your call", "drops one single call — long or short" |
| **ja** 日本語 | 일본 | 크립토 트위터 말투 + **문화 현지화** | 김프→円安, 흑우→カモ, 김치코인→草コイン, 초록불→青天井. 캔들 색 관용구는 한·일 반대(빨강↑/파랑↓)라 표현 조정(溶ける準備 등). 브랜드 タコちゃん |
| **ru** Русский | 러시아 | 러시아 크립토 커뮤니티 구어체 | 자연 번역 + 현지 밈 |
| **uk** Українська | 우크라이나 | 우크라이나 크립토 커뮤니티 | 자연 번역 + 현지 밈 |
| **tr** Türkçe | 터키(크립토 초강세) | 터키 크립토 커뮤니티 슬랭 | 자연 번역 + 현지 밈 |
| **id** Bahasa Indonesia | 인도네시아 | 인니 크립토 커뮤니티 | 자연 번역 + 현지 밈 |
| **vi** Tiếng Việt | 베트남 | 베트남 크립토 커뮤니티 슬랭 | "cá lẹp"(잔챙이), "múc không suy nghĩ"(생각없이 매수) |
| **hi** हिन्दी | 인도 | 힌디+영어 혼용(Hinglish) | "पंप", "टू द मून", "बड़े खिलाड़ी"(큰손) |
| **pt** Português | 브라질(크립토 대국) | 브라질 포르투갈어 크립토 슬랭 | "formiguinha"(개미), "manada"(군중), "rug pull" |
| **es** Español | 라틴아메리카·스페인 | LatAm 크립토 슬랭 | "pez pequeño"(잔챙이), "a la luna", "aguante ganador" |

> 나이지리아·필리핀 등 영어권 크립토 커뮤니티는 **en**으로 커버, 인도는 **hi**(Hinglish) 별도 제공.

## 4. 번역 커버리지 (현재)

| 영역 | 커버 |
|---|---|
| 헤더·푸터·언어스위처·메타 제목/설명 (common) | **11개 언어 전부** |
| 예언 UI·쿨다운·바로받기·결과 (oracle) | **11개 언어 전부** |
| 페이지 본문(시장온도·소개·예언하기·개인정보·문의·면책 등) (pages) | **11개 언어 전부** |
| 재미 문구·로딩·결과 키워드 풀 (fun) | **11개 언어 전부** |
| 가이드 글 10편 | **ko·en**(전편), **ja**(일부) — 나머지 언어는 **ko 폴백** |

## 5. 유지보수 방법

- **UI/문구 수정**: `messages/{locale}/{namespace}.json` 의 해당 키 수정. ko에만 있는 키는 다른 언어에서 자동으로 ko 폴백.
- **새 언어 추가**: `i18n/routing.ts`의 `locales`에 추가 → `components/LangSwitcher.tsx` 표시명 추가 → `messages/{새locale}/` 에 네임스페이스 파일 생성(없으면 ko 폴백으로도 동작).
- **가이드 번역 추가**: `content/guide/{locale}/{slug}.md` 생성(frontmatter title/description 포함).

## 6. 품질 참고

- **en·ja·ru·es·pt**: 커뮤니티 구어체 품질 양호.
- **vi·hi·uk·id·tr**: 최대한 자연스럽게 작성했으나 **원어민 검수 1회 권장**(특히 fun의 슬랭/밈).
- 나머지 언어 **가이드 글**은 아직 ko 폴백 — 필요 시 언어별로 순차 번역 예정.
