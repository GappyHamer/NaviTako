# [클로드 코드 실행 프롬프트] 롱숏 신탁 — BTC 선물 롱/숏 재미 추천 사이트

너는 이 프로젝트의 시니어 풀스택 개발자다. 아래 명세대로 **Phase 1(MVP)** 을 이 폴더에 구현하라.
사용자는 웹 개발 경험이 없으므로: 결정이 필요하면 스스로 합리적으로 결정하고, 작업 후에는 "무엇을 했고, 사용자가 직접 해야 할 일(배포 클릭 등)이 무엇인지"를 쉬운 말로 요약하라.

## 1. 프로젝트 개요

- **이름(가칭)**: 롱숏 신탁 — 버튼을 누르면 점쟁이 문어가 BTC 선물 **롱/숏 중 하나를 "신탁"으로 내려주는** 오락 사이트
- **핵심 규칙**: 순수 랜덤이 아니다. 실제 시장 지표로 확률을 기울이고, **그 오차 범위 안에서만 랜덤**으로 결정한다 (아래 3장 알고리즘 고정 스펙).
- **절대 원칙**: 구체적 수치·목표가·진입가 제시 금지. 결과는 오직 "롱" 또는 "숏" + 재미 멘트. 투자 조언처럼 보이는 표현("시그널", "추천 진입" 등) 금지 — "신탁", "점괘" 등 오락 어휘만 사용.
- **수익화 전제**: 구글 애드센스 심사를 통과할 수 있는 구조(콘텐츠 페이지, 필수 페이지, SEO)를 처음부터 포함한다.

## 2. 기술 스택 (고정 — 변경 금지)

- Next.js 최신 안정 버전(App Router) + TypeScript + Tailwind CSS
- 배포: Vercel 무료 플랜. DB 없음(Phase 1), 로그인 없음, API 키 필요한 외부 서비스 사용 금지
- 다크 테마 기본(크립토 감성), 모바일 퍼스트 반응형

## 3. 핵심 기능 스펙

### 3.1 데이터 수집 (서버 전용)

Route Handler `/api/market` 하나가 아래 5개를 **서버에서** 병렬 호출하고, 결과를 **60초 캐시**한다(`revalidate` 또는 `unstable_cache`). 클라이언트가 외부 API를 직접 호출하는 것 금지.

| 키 | 엔드포인트 | 사용 값 |
|---|---|---|
| fng | `https://api.alternative.me/fng/?limit=1` | `data[0].value` (0~100) |
| funding | `https://fapi.binance.com/fapi/v1/premiumIndex?symbol=BTCUSDT` | `lastFundingRate` (소수) |
| topRatio | `https://fapi.binance.com/futures/data/topLongShortPositionRatio?symbol=BTCUSDT&period=1h&limit=1` | `longShortRatio` |
| globalRatio | `https://fapi.binance.com/futures/data/globalLongShortAccountRatio?symbol=BTCUSDT&period=1h&limit=1` | `longShortRatio` |
| ticker | `https://fapi.binance.com/fapi/v1/ticker/24hr?symbol=BTCUSDT` | `priceChangePercent`, `lastPrice` |

**장애 내성(중요)**: 각 호출은 개별 try/catch + 3초 타임아웃. 실패한 지표는 제외하고 성공한 지표만으로 계산한다. **전부 실패하면** P(롱)=0.5로 동작하고 UI에 "오늘은 순수 운빨 모드 🎲" 배지를 띄운다. 어떤 경우에도 버튼은 항상 동작해야 한다.

### 3.2 신탁 알고리즘 (고정 스펙 — `lib/oracle.ts` + 가중치는 `config/oracle.config.ts` 분리)

각 지표를 롱 편향 +1 ~ 숏 편향 -1로 정규화 (`clamp`는 [-1,1] 제한):

```
s1 공포탐욕(역발상)   = (50 - fng) / 50           // 공포일수록 롱
s2 펀딩비(역발상)     = -clamp(rate / 0.0005)      // 펀딩 과열(+0.05%)이면 숏
s3 고수 포지션(추종)  = clamp(ln(topRatio) / ln(2)) // 고수가 롱이면 롱
s4 전체 계정(역발상)  = -clamp(ln(globalRatio) / ln(2)) // 개미 쏠림 반대
s5 24h 모멘텀(추종)   = clamp(pct / 5)              // ±5%에서 포화

가중치 w = { s1: 0.25, s2: 0.20, s3: 0.25, s4: 0.10, s5: 0.20 }
S = Σ wᵢ·sᵢ   (실패 지표는 제외하고 남은 가중치 합으로 재정규화)
P(롱) = clamp(0.5 + 0.35·S, 0.15, 0.85)   // 어느 쪽도 확정 금지 = "오차 범위 내 랜덤"
결과 = Math.random() < P(롱) ? "LONG" : "SHORT"
```

- 계산은 **클라이언트에서** 수행(`/api/market` 응답을 받아서). 클릭마다 서버 호출이 늘지 않아 무료 티어로 충분해진다.
- P값·지표 수치는 결과 화면에 **표시하지 않는다**. 대신 S 구간별 "시장 기류" 문구만: S>0.3 "위로 끌어당기는 기류", 0.1~0.3 "롱 쪽으로 살짝 기움", -0.1~0.1 "팽팽한 줄다리기", -0.3~-0.1 "숏 쪽으로 살짝 기움", S<-0.3 "아래로 끌어당기는 기류".

### 3.3 UI/UX

- **메인 화면**: 문어 캐릭터(이모지 🐙 + CSS 애니메이션으로 시작, 이미지 에셋은 나중에 교체 가능하게) + 큰 신탁 버튼 하나. 그 외 요소 최소화.
- **연출**: 버튼 클릭 → 2~3초 두근두근 연출(문어 흔들림, "지표를 읽는 중...", "고수들의 포지션을 훔쳐보는 중..." 같은 로딩 문구 순환) → 카드 뒤집기로 결과 공개.
- **결과 카드**: LONG(초록)/SHORT(빨강) 대형 표기 + 시장 기류 문구 + 재미 멘트 1개(롱용/숏용 각 20개 이상 풀에서 랜덤, 밈 감성으로 작성. 예: "문어는 이미 롱을 잡았습니다. 발이 8개라 8배는 아닙니다.") + 면책 한 줄 + [다시 뽑기] [결과 공유] 버튼.
- **공유**: Web Share API(모바일) + 클립보드 복사 폴백. 공유 URL `/result/[long|short]`는 결과별 **동적 OG 이미지**(`next/og` ImageResponse)로 SNS·카톡 미리보기가 예쁘게 나오게.
- **탭(상단 네비)**: 신탁(홈) / 시장온도 / 가이드 / 소개.

### 3.4 페이지 구성

1. `/` 홈(신탁)
2. `/market` 시장온도 — `/api/market` 지표를 게이지·이모지 시각화(수치는 여기선 표시 OK, 교육 목적 프레이밍). 공포탐욕지수는 출처(alternative.me) 표기.
3. `/guide` 콘텐츠 허브 + 하위 글 **10개** (애드센스 심사용 핵심). 각 글 한국어 1,800자 이상, markdown/MDX 관리:
   공포탐욕지수란 / 펀딩비란 / 롱숏비율 읽는 법 / 선물거래와 청산 위험 / 레버리지의 수학 / 김치프리미엄이란 / RSI 기초 / 비트코인 도미넌스 / 왜 개미는 항상 물릴까(행동경제학) / **이 사이트 알고리즘 전체 공개**(차별화 포인트)
   모든 글 끝에 "본 콘텐츠는 교육·오락 목적이며 투자 조언이 아닙니다" 삽입.
4. `/about` 소개, `/privacy` 개인정보처리방침(GA·애드센스 쿠키 언급 포함), `/contact` 문의(이메일 표기), `/disclaimer` 면책조항 — 애드센스 필수 4종.
5. 푸터: 면책 한 줄 + 위 페이지 링크 상시 노출.

### 3.5 면책 문구 (원문 그대로 사용)

## 4. 비기능 요구사항

- **Vercel 리전(치명적 함정)**: Binance API는 미국 IP를 차단(451)한다. `vercel.json`에 `{ "regions": ["icn1"] }`(서울)를 반드시 포함하고, README에 이 이유를 기록하라.
- **보안**: `next.config`에 보안 헤더(X-Frame-Options: DENY, X-Content-Type-Options: nosniff, Referrer-Policy: strict-origin-when-cross-origin, 합리적 CSP). 외부 API는 서버 전용. 시크릿·환경변수 없이 동작(v1). 의존성은 최소로.
- **SEO**: 페이지별 metadata, OG 태그, sitemap.xml, robots.txt, JSON-LD(WebSite), 시맨틱 HTML. 타이틀 예: "롱숏 신탁 — 재미로 보는 비트코인 롱/숏 점괘".
- **애드센스 대비**: ① 광고 위치용 `<AdSlot />` 빈 컴포넌트(가이드 글 상/중/하, 시장온도 하단 — **신탁 버튼 근처는 금지**) ② `public/ads.txt` 자리(주석으로 발급 후 교체 안내) ③ 레이아웃이 광고 삽입 시에도 CLS가 안 생기게 고정 높이 예약.
- **품질 기준**: `npm run build` 무경고 통과, Lighthouse(모바일) Performance·SEO 90+, 문어 연출 포함 60fps 체감.

## 5. 작업 순서 (Phase 1)
0. 작업 전 예상 가용 토큰 및 작업 시간 파악 후 보고 하고 작업 시작. 
1. Next.js 프로젝트 스캐폴드 + Tailwind + 폴더 구조(`lib/`, `config/`, `content/guide/`)
2. `/api/market` + 장애 내성 + 캐싱 → 3. `lib/oracle.ts` 알고리즘 + 단위 테스트(가중치 재정규화, clamp, 전멸 폴백 케이스) → 4. 홈 UI·연출·멘트 풀 → 5. 공유 OG → 6. 나머지 페이지·가이드 글 10개 초안 → 7. SEO·보안 헤더·ads.txt·vercel.json → 8. 빌드 검증 + 로컬 실행법·Vercel 배포법을 왕초보 기준 단계별로 `DEPLOY.md`에 작성

완료 후 보고: 실행 방법(`npm run dev`), 배포 시 사용자가 클릭해야 할 것, 도메인 연결 후 해야 할 것(서치콘솔, 애드센스 신청 시점)을 요약할 것.

## 6. Phase 2·3 (지금 구현하지 말 것 — 사용자가 별도 지시할 때만)

- **Phase 2 통계**: Upstash Redis(무료)로 일별 롱/숏 뽑기 카운트 저장 → "오늘 문어의 신탁 분포 63:37" 표시. Vercel Cron으로 신탁 시점 가격 대비 4h/24h 등락을 기록해 "문어 적중률" 공개(재방문 루프 핵심).
- **Phase 3 채팅**: Supabase Realtime 익명 닉네임 채팅 탭. 필수: 메시지 길이 제한, 초당 전송 제한, 금칙어 필터, 링크 자동 차단, 신고 버튼. XSS 방지(모든 입력 이스케이프).
