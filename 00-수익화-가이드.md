## 0. 한눈에 보기

| 항목 | 결정 사항 |
|---|---|
| 컨셉 | 데이터 기반 "운세 토이" — 시장 지표로 확률을 기울인 뒤 그 오차 범위 안에서 랜덤 추천 |

## 1. 컨셉과 차별화 전략

### 레퍼런스 분석

| 사이트 | 정체 | 수익 모델 | 배울 점 |
|---|---|---|---|
| 김프가 | 김프 실시간 대시보드 + 커뮤니티/채팅/뉴스 | 배너, 제휴 | 채팅으로 체류시간 확보, 지표 탭 구조 |
| 더따리 | 김프 테이블 | **거래소 페이백(레퍼럴)** 전면 배치 | 이 계열의 진짜 수익은 레퍼럴이라는 것 |
| 크라이프라이스 | 김프 대시보드 | 배너 | 김프 시장은 이미 포화 |
| OshiZi | 재미 도구·게임 스튜디오 | **애드센스** (승인 이력 확인됨) | 재미+공유로 월 활성 10만 달성, 블로그로 SEO 유입 |

핵심 통찰: 김프 사이트들은 **정보 조회** 사이트라 서로 대체 가능하고 경쟁이 포화 상태입니다. OshiZi는 **참여형 재미**로 성공했습니다. 우리는 김프 사이트를 따라하는 게 아니라 **"크립토 감성의 참여형 토이"** 포지션을 잡습니다.

### 차별화 포인트 5가지

1. **근거 있는 랜덤** — 단순 코인 던지기가 아니라 실제 지표(펀딩비, 고수 포지션, 공포탐욕지수 등)로 확률을 기울임. "알고리즘 설명 페이지"로 투명하게 공개하면 재미+신뢰+콘텐츠(SEO)가 동시에 해결됨. 이 조합의 사이트는 현재 없음.
2. **캐릭터 연출** — 점쟁이 문어(월드컵 점쟁이 문어 '파울' 오마주) 같은 마스코트가 "신탁"을 내리는 연출. 결과가 밈이 되어 공유됨.
3. **공유 루프** — 결과마다 전용 공유 카드(OG 이미지) 생성. 커뮤니티(디시, 코인판, X)에 결과 스크린샷이 돌게 설계. OshiZi 성장 공식.
4. **재방문 루프** — "오늘의 롱/숏 분포", "문어 적중률 추적"(2단계) 같은 매일 바뀌는 통계. 김프 확인하듯 매일 들르게 만듦.
5. **커뮤니티** — 익명 채팅 탭(3단계). 김프가의 채팅이 증명한 체류시간 장치.

### 네이밍 후보 (도메인 가용성 확인 필요)

- **롱숏문어** (longshort-octopus / muneo.gg 계열) — 캐릭터 중심, 밈 확장성 최고
- **롱숏신탁** — "신탁" 컨셉이 연출과 잘 붙음
- **가즈아오라클** — 커뮤니티 감성
- 추천: 캐릭터가 있는 1번. 사이트 이름보다 "문어가 롱이래"라는 문장이 퍼지는 게 목표.

## 2. 기술 스택과 데이터

### 왜 이 스택인가 (첫 프로젝트 기준)

- **Next.js + Vercel**: 배포가 버튼 하나, HTTPS 자동(애드센스 필수 조건), 무료 티어로 충분, 한국어 자료 최다
- **1단계는 DB 없음**: 로그인·저장 없는 정적+API 구조 → 보안 공격면 자체가 거의 없음
- 채팅(3단계)만 Supabase 무료 티어 사용

### 데이터 소스 (전부 무료·키 불필요, 검증 완료)

| 지표 | 엔드포인트 |
|---|---|
| 공포탐욕지수 | `https://api.alternative.me/fng/?limit=1` (출처 표기 필요) |
| 펀딩비 | `https://fapi.binance.com/fapi/v1/premiumIndex?symbol=BTCUSDT` |
| 고수 포지션 롱숏비 | `https://fapi.binance.com/futures/data/topLongShortPositionRatio?symbol=BTCUSDT&period=1h&limit=1` |
| 전체 계정 롱숏비 | `https://fapi.binance.com/futures/data/globalLongShortAccountRatio?symbol=BTCUSDT&period=1h&limit=1` |
| 24h 변동률 | `https://fapi.binance.com/fapi/v1/ticker/24hr?symbol=BTCUSDT` |

### ⚠️ 반드시 알아야 할 함정 1가지

**Binance API는 미국 IP를 차단(HTTP 451)합니다.** Vercel 서버 함수의 기본 리전이 미국(iad1)이라 그대로 배포하면 Binance 호출이 전부 실패합니다.
→ 해결: 프로젝트에 `vercel.json`으로 함수 리전을 **서울(icn1)** 로 지정. (프롬프트에 이미 포함해 뒀습니다)
→ 이중 안전장치: 일부 지표 호출이 실패해도 나머지 지표만으로 동작하는 폴백 로직 포함.

## 3. 로드맵

**1주차 — MVP**: 클로드 코드로 개발(프롬프트 파일 사용) → Vercel 배포 → 지인 테스트
**2주차 — 애드센스 준비**: 도메인 구입·연결(가비아/클라우드플레어, 연 1.5~2만 원) → 콘텐츠 글 10개+ 다듬기(프롬프트가 초안 생성, 아래 4장 참고) → 통계 탭 완성
**3주차 — 애드센스 신청**: 체크리스트(아래) 확인 후 신청 → 심사 수일~4주 대기 → 대기 중 커뮤니티에 사이트 공유 시작(트래픽 실적 쌓기)
**4주차~ — 확장**: 채팅 탭(Supabase) → 적중률 추적 → 거래소 레퍼럴 배치 → 카카오 애드핏 병행

## 4. 수익화 계획

### 애드센스 승인 체크리스트 (2026년 기준)

- [ ] 본인 도메인 + HTTPS (Vercel이 HTTPS 자동 처리)
- [ ] 콘텐츠 글 **10~15개 이상, 각 1,500자 이상** — 도구만 있는 사이트는 "가치 없는 콘텐츠"로 거절되는 게 최근 추세. 지표 해설 글이 핵심 방어선
- [ ] **소개 / 개인정보처리방침 / 문의** 페이지 (없으면 자동 탈락 수준의 필수 요소)
- [ ] 면책조항 페이지 (크립토 특성상 강력 권장)
- [ ] sitemap.xml, robots.txt, 서치콘솔 등록
- [ ] AI 생성 글 그대로 올리지 말 것 — 초안을 직접 다듬어서 게시 (설치된 `휴머나이즈` 스킬로 다듬은 뒤 본인 검수 추천)

### 크립토 콘텐츠 주의사항

구글 정책상 크립토 **교육·정보 콘텐츠는 허용**되지만, **투자 자문·시그널·수익 보장** 성격은 제재 대상입니다. 따라서:

- 사이트 전체 톤을 "오락"으로 유지 — "추천", "시그널", "자문", "수익" 같은 단어 대신 "신탁", "점괘", "재미로 보는" 사용
- 모든 결과 화면과 푸터에 면책 문구 상시 노출 (프롬프트에 문구 포함됨)

### 광고 배치 원칙

- 심사 통과 전: 광고 코드 없이 깔끔한 상태로 신청
- 통과 후: 콘텐츠 글 페이지 중심 배치(수익 대부분이 여기서 발생), 모바일 앵커 광고 1개
- **메인 버튼 주변에는 광고 금지** — 실수 클릭 유도로 계정 정지 사유가 됨
- ads.txt 파일 필수 (프롬프트에 자리 마련됨)

### 현실적인 기대치와 진짜 수익원

- 애드센스 수익은 한국 트래픽 기준 페이지뷰 1,000당 대략 1,000~4,000원 수준. 월 10만 뷰가 나와야 월 십수만 원 규모 → **애드센스는 기본기, 대박 수단이 아님**
- 이 계열 사이트의 실질 수익원은 **거래소 레퍼럴**(더따리가 "페이백"을 전면에 내세우는 이유). 트래픽이 잡히면 Bitget·Bybit 등 레퍼럴 프로그램 가입 후 "수수료 할인 링크" 배치 — 가입 1건이 애드센스 수천 클릭 가치
- 보조: 카카오 애드핏(승인 쉬움, 애드센스 대기 중에도 부착 가능)

## 5. 보안 (프롬프트에 이미 반영됨)

1단계는 로그인·DB·개인정보가 없어서 공격면이 최소입니다. 그래도 다음을 프롬프트에 포함했습니다:
보안 헤더(CSP, X-Frame-Options 등), 외부 API는 서버에서만 호출(60초 캐시로 폭주 방지), 의존성 최소화, 비밀키 없음 원칙(v1), 채팅 단계에서는 입력 검증·금칙어·속도 제한 추가.

## 6. 법적 유의사항

- 결과 화면·푸터에 "투자 권유가 아닌 오락용" 면책 상시 노출 (문구는 프롬프트에 포함)
- 유료 구독·리딩방 형태로 확장하는 순간 유사투자자문 규제 이슈가 생길 수 있으니 **무료+광고 모델 유지**가 안전
- 이 문서는 법률 자문이 아니므로, 유료화 등 확장 시점에는 전문가 확인 권장
