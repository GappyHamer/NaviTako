# 🔐 구글 로그인 연결 가이드

코드는 이미 들어가 있습니다. **무료**이며, Google OAuth 앱을 만들어 환경변수 3개만
Vercel에 넣으면 켜집니다. 환경변수가 없으면 로그인 버튼이 숨겨지고 기존 익명 방식이
그대로 동작하니, 설정 전에도 사이트는 멀쩡합니다.

비용: **월 0원** (구글 OAuth 무료 + 자체 구현이라 유료 인증 서비스 불필요).

---

## 1단계. Google Cloud에서 OAuth 앱 만들기

1. https://console.cloud.google.com → 상단에서 **프로젝트 만들기**(아무 이름, 예: navitako)
2. 왼쪽 메뉴 **API 및 서비스 → OAuth 동의 화면**
   - User Type: **외부(External)** 선택 → 만들기
   - 앱 이름(예: 롱숏 예언), 지원 이메일(본인), 개발자 연락처(본인 이메일) 입력 → 저장
   - "테스트 사용자"에 본인 구글 이메일 추가(게시 전 테스트용). 나중에 **게시(Publish)** 하면 누구나 로그인 가능.
3. 왼쪽 메뉴 **사용자 인증 정보 → 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
   - 애플리케이션 유형: **웹 애플리케이션**
   - **승인된 리디렉션 URI**에 아래를 추가(현재 배포 주소 기준):
     ```
     https://navi-tako.vercel.app/api/auth/callback
     ```
     (커스텀 도메인 연결했으면 `https://내도메인.com/api/auth/callback` 도 함께 추가)
   - 만들기 → **클라이언트 ID**와 **클라이언트 보안 비밀**이 나옵니다. 복사해 두세요.

---

## 2단계. Vercel에 환경변수 3개 추가

Vercel → navi-tako → **Settings → Environment Variables** 에서 아래 3개 추가(All Environments):

| 이름 | 값 |
|---|---|
| `GOOGLE_CLIENT_ID` | 1단계에서 받은 클라이언트 ID |
| `GOOGLE_CLIENT_SECRET` | 1단계에서 받은 클라이언트 보안 비밀 |
| `AUTH_SECRET` | 아무 긴 랜덤 문자열(세션 쿠키 서명용). 예: 아래 명령으로 생성 |

`AUTH_SECRET` 생성(둘 중 편한 것):
- 터미널: `openssl rand -base64 32`
- 또는 https://generate-secret.vercel.app/32 에서 복사

추가 후 **Deployments → 맨 위 ⋯ → Redeploy**.

---

## 3단계. 동작 확인

1. 재배포 후 `navi-tako.vercel.app/predict` 접속
2. "구글로 로그인" 버튼 클릭 → 구글 계정 선택 → 다시 예측 페이지로 돌아오면 성공
3. 로그인하면 리더보드 기록이 **구글 계정 기준**으로 저장돼, 다른 기기에서 로그인해도 이어집니다. (로그인 전 익명 기록은 익명 상태로 남습니다.)

---

## 자주 묻는 것

- **redirect_uri_mismatch 에러**: 1단계의 승인된 리디렉션 URI가 실제 접속 주소와 정확히 일치해야 합니다(끝의 `/api/auth/callback` 포함, http/https·도메인 철자까지). 도메인 바꾸면 여기도 추가하세요.
- **"앱이 확인되지 않음" 경고**: OAuth 동의 화면을 아직 게시(Publish) 안 했거나 심사 전이면 뜹니다. 테스트 사용자로 등록한 계정은 "고급 → 이동"으로 진행 가능. 정식 운영하려면 동의 화면을 게시하세요(간단한 앱은 별도 심사 없이 게시됨).
- **월 비용**: 없음. 구글 로그인은 사용자 수와 무관하게 무료입니다.
